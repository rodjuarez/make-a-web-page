Lesson 1: Recap

Introductory Concepts
What is the Web?

The Web is the collection of all the HTML documents that exist in the largest network, which is a series of servers.

What is HTML?

HTML stands for Hyper Text Markup Language, which is the language that browsers interpret as what we see as pages in the Internet. At the end it is a programming language. It is estimated that around 30 billion HTML pages exist on the Internet.

What are URLs?

It stands for Uniform Resource Locator and it is the standard to request information and locate HTML pages on the Internet. A URL is sent to the server that stores the page requested and the server sends back that particular page to be interpreted by the browser of the individual computer where the request was sent.

What is HTTP?

It stands for Hyper Text Transfer Protocol, which is the protocol, or series of standardize instructions that the Internet uses to link HTML pages.

Web Applications

How does the Internet Works?

User sends a request from a device
The URL requested goes to the Internet
Through the HTTP protocol it is located in a server
The server sends the requested URL through HTTP
The URL travels through the Internet
The link is accomplished through the user’s browser which interprets the HTML Page for the user to read/see/listen.

What is a Tag?

A Tag is an HTML label describing the content of an HTML page. Tags are always contained within a less than and a more than brackets. And in most cases are used in pairs, an opening tag “(tag)” and a closing tag “(/tag)".

What is a Void Tag?

A Void Tag has no content and therefore does not require a closing tag. Examples of a void tag are the bracket tag “(br), and image tag “(img)”.

What is an Inline Tag?

Inline Tags are simple line tags. That is, they are not contained in boxes. Examples are: the (br), (img), (em), (b) tags, etc.

What is a Block Tag?

Block Tags are always contained in boxes (although not always seen as such through their representation of the content. Examples are: The paragraph tag “(p)(/p)”, the anchor tag “(a)(/a)”, or the division tag “(div)(/div)”.

What is an Element?

An Element is everything that is contained between an opening and a closing tag.

What is an Attribute?

Attributes are properties assigned to the content in a tag, such as color, font, size, location, border size, etc.

Lesson 3 – Adding CSS Style to the HTML Structure

CSS, or Cascading Style Sheets

CSS provides a way to avoid repetition, and an easier way to change the attributes of like elements at once providing consistency. The cleaner the code is (less repetition) the easier it will be to follow and the faster it will load.

Coding should always be ”structured”, meaning that divisions (div) tags have been used to separate logically combined elements into blocks for easier manipulation, understanding, and debugging, thus the term ”Cascading”, or

The term ”Styles” term comes from the fact that the elements in the HTML coding can be manipulated with different “selectors” such as font, size, color, etc. These styles can be specified to particular elements, but CSS Styles are better applied when similar elements are manipulated at once by these selectors, such as headers, content, titles, etc.

The term ”Sheets” refers to the fact that many different set of CSS coding can be defined for one HTML document, each one having unique coding to different parts of the HTML page. However, one must be careful not to create many CSS sheets when the code could be included in only one. It is important to point out that changing the visual properties of an element only change the way that they are displayed by the browser, and not the type of element that it is.

Inheritance

CSS has a key feature called “Inheritance”, which means that the properties are not just applied to a specific element, but also to its descendant elements on a tree structure, in other words, according to its hierarchy, also called nesting. However, two things must be remembered:
1. The most descriptive style is the one that overrules the style.
2. Box related properties are not inherited
The properties that can be inherited are: color, font, letter spacing, line height, list style, text align, text indent, text transform, visibility, white space and word spacing.
The properties that cannot be inherited are: background, border, display, float and clear, height and width, margin, min and max height and width, outline, overflow, padding, position, text decoration, vertical align and z-index.
Inheritance reduces the amount of code that developers need to write.

Selectors Coding

A “Selector” specifies the element or like elements that it will modify. The coding of the selector contains the element, a left curly bracket on the first line. On the second line an attribute followed by a colon, a value followed by a semi colon on the second line. And a right curly bracket on line three, closing the selector.
An example would be:
.content {
color: green;
}

Rules

Everything that is contained within the curly brackets is called a rule. The rule is applied to the element.

Adding CSS into a Web Page

There are 3 methods to add CSS into a Web Page:
1) Adding the CSS code into the Head section of the HTML page. Good for small projects.
2) Writing the CSS code into a separate file and linking it to the Web page. The link should be placed within the Head section of the HTML page with come such as: “”. This is particularly more efficient for larger projects.
3) Writing the CSS code inline with the HTML code. This is the least effective method as it creates repetition and thus inefficient coding, larger lines of code, and slower processing, as well as the opportunity for more mistakes.

Properly coded these comments are not displayed by the browser, so they can only be seen through the coding.
Example:
Comments in HTML

Boxes

In HTML all elements are boxes, even as they are not necessarily displayed by the browsers. Each box, going from the core out is formed by the content at its core, then surrounded by the padding which is a clear space around the content that avoids overlapping and that is affected by the background color of the box. The padding is surrounded by a border which inherits from the box too, and the border is surrounded by a margin which is the transparent outer space of the box.

To calculate the size of a box one must add the width of the content, plus both sizes of the padding, plus the size of both sides of the border. Sizing these boxes in a precise manner is complicated. There are 3 ways to do this:
1) You can specify the size of the box in pixels which is a hard form of sizing them and they never change in size.
2) You can specify the size of the box in percentage which will resize the box depending on the size in which the page is being viewed.
3) You can mix the prior two ways. You could, for example, specify a max or minimum size in a flex resizing manner.

The flex sizing option should be added as a rule in the following manner: display: flex;.

Coding CSS and Browser Differences

Because there are differences in the way different browsers display HTML and CSS, in order to ensure that the viewer will see our coding the way we intended it to be seen, the following rule needs to be added in CSS:
*{
-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
}
